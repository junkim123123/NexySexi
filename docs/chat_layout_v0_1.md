# NexSupply Chat-First UX 설계 v0.1

## 0. 문서 목적 / 메타

이 문서는 NexSupply의 채팅 중심(Chat-First) 인터페이스 초안 설계안이다.

### 목표

기존 "폼 입력 → 결과" 구조를
**"Nexi와 대화하면서 자연스럽게 정보를 모으고, 분석/프로젝트로 이어지는 흐름"**으로 전환하기 위함.

이 버전은 기획/피드백용이며, 실제 컴포넌트/코드 구현 전에 기준점 역할을 한다.

---

## 1. 제품 및 맥락 개요

### 서비스 한 줄 정의

> "NexSupply는 AI와 현지 인프라를 결합해, 중소 셀러/리테일러가 '한 박스'부터 안전하게 해외 소싱을 시작할 수 있게 해주는 매니지드 소싱 플랫폼이다."

### 왜 채팅 기반으로 바꾸는가

1. **유저가 "폼 작성"보다 "상담받는 느낌"을 훨씬 편하게 느낀다.**
2. **Dr.B 스타일처럼 친밀감 + 집중감 + '나만 보고 있다' 느낌을 주기 위함.**
3. **복잡한 질문 세트를 한 번에 던지는 대신, 대화형으로 쪼개서 피로도를 줄이기 위함.**

---

## 2. Nexi 페르소나 정의

### 역할

- NexSupply의 가상 소싱 매니저 / PM
- 유저의 프로젝트를 "설계하고 정리해주는 사람" 느낌.

### 톤 & 말투

친근하지만, 덜 가벼운 "현실적인 친구 + 운영 매니저"

**예시**:
- "오케이, 대충 그림 잡혔어요. 이제 단가 쪽을 좀 더 디테일하게 볼게요."
- "지금 듣기로는 '아마존 FBA 테스트'에 딱 맞는 프로젝트 같아요."

### 알고 있는 것

- FBA / DTC / 도매 구조
- FOB, DDP, HS Code, MOQ, 리드타임 등 기본 개념

### 경계선 (하지 말 것)

- ❌ 확신 없는 숫자를 "보장"하는 말투 금지
- ❌ 법적/세무적 조언인 것처럼 들리는 표현 피하기

## 3. 핵심 유저 여정 (High-Level Flow)

### 입장
웹앱 접속 → 채팅 화면에서 Nexi 첫 인사

### 프로젝트 프레이밍
"어떤 채널에서 팔 건지 / 어느 나라로 / 어떤 제품류인지"를 가볍게 정의

### 핵심 정보 수집
제품 정보(링크/이미지/설명), 목표 수량, 타임라인, 예산 레인지

### 분석 준비 완료
Nexi가 "지금 정보로도 1차 추정 리포트는 가능"이라는 신호를 줌

**CTA**: "1차 DDP / 리스크 요약 받아보기" 버튼

### 결과 공유
채팅 안에서 요약 카드 + 핵심 숫자

자세한 리포트는 별도 페이지/모달로 연결

### 후속 액션
"이 프로젝트 진짜로 진행해 볼까요?"

**CTA**: $10 Deposit 걸고 전문가 팀 연결 or "조금 더 수정해볼게요"

## 4. 채팅 플로우 상세

### 4.1 첫 인사 & 목적 파악

**Nexi 기본 인사 + 짧은 설명**:

> "여기는 NexSupply의 Nexi예요. 한 박스부터 시작하는 해외 소싱 설계 도와드릴게요."

**빠른 분류 질문 (버튼/칩 사용)**:

> "어디에서 팔 계획이세요?"

**옵션**:
- [Amazon FBA]
- [Shopify / DTC]
- [오프라인 리테일]
- [아직 확실치 않음]

### 4.2 프로젝트 타입 & 채널

선택에 따라 후속 질문 브랜칭

**예: Amazon FBA 선택 시**:

> "이미 FBA 경험 있으세요?"

**옵션**:
- [첫 FBA 도전]
- [이미 몇 개 제품 운영 중]
- [대행사/에이전시 사용 중]

### 4.3 목표/타임라인/예산

**질문 예시 (칩 + 자유 입력 혼합)**:

**목표 질문**:
> "목표는 어떤 느낌인가요?"

**옵션**:
- [테스트용 소량]
- [꾸준한 리스팅 하나 더 추가]
- [기존 상품 대체/스위칭]

**타임라인 질문**:
> "언제쯤 판매 시작하면 좋겠어요?"

**옵션**:
- [1~2개월 안]
- [3개월 이상 괜찮음]
- [정해진 데드라인 있음]

### 4.4 제품 정보 수집

**허용 입력 형태**:
- 링크(아마존, 자사몰, 경쟁사)
- 간단 텍스트 설명
- 이미지 업로드 (추후)

**Nexi 리액션**:
- "오, 이 스타일이군요. 비슷한 공장들은 중국/한국에 꽤 있어요."
- "혹시 꼭 지켜야 하는 스펙이 있나요? (예: 무설탕, 글루텐 프리 등)"

### 4.5 분석 요청 단계

**Nexi 정리 멘트**:

> "여기까지 들은 걸로 1차 DDP / 리스크 추정은 가능해요."
> 
> "완전 확정 견적은 아니고, '이 정도면 할 만하다 / 아니다'를 보는 단계라고 생각하시면 돼요."

**CTA 버튼**:
- [1차 분석 시작하기]
- [정보 조금 더 추가하고 싶어요]

### 4.6 결과 전달 & 후속대화

**채팅 메시지로 요약**:

> "대략 DDP 단가는 X~Y USD/개 정도로 예상되고, 관세 리스크는 중간 정도예요."

**추가 질문 안내**:

> "어느 부분이 가장 궁금하세요?"

**옵션**:
- [DDP 단가 구조]
- [관세/규제]
- [MOQ/리드타임]
- [포장/라벨링]

## 5. UI 레이아웃 구조

### 상단 헤더

- NexSupply 로고
- 현재 프로젝트 이름/태그
- 간단한 진행 상태 표시 (예: "정보 수집 중 / 1차 분석 완료")

### 중앙 메인 영역

**채팅 타임라인**:
- Nexi 버블 (왼쪽 정렬, subtle 배경)
- 유저 버블 (오른쪽 정렬, primary 배경)
- 임베디드 카드 (분석 결과, CTA 등)

### 우측 사이드 패널 (옵션)

**현재까지 정리된 프로젝트 스냅샷**:
- 제품 요약
- 채널 / 타겟 국가
- 예상 DDP 레인지(있다면)
- "이번 대화에서 이미 채워진 필드" 리스트

**NOTE**: MVP에서 포함할지, 나중에 추가할지 결정 필요

### 하단 입력 영역

- 텍스트 입력창
- 바로 위에 Quick Reply 칩/버튼
- 우측에 보조 CTA (예: "지금까지 내용 정리 보기")
- 이미지 업로드 버튼 (옵션)

## 6. 버튼 / 칩 패턴

### Quick Reply 칩

- 예/아니오
- 단일 선택 (예: 채널 선택)
- **"잘 모르겠어요" 옵션 항상 존재**

### 메인 CTA 버튼

- "1차 분석 시작하기"
- "결과 다시 업데이트하기"
- "$10 Deposit 걸고 프로젝트 시작"

### 세컨더리 액션

- "나중에 할래요"
- "이 프로젝트 저장만 해둘게요"

## 7. 상태 / 로딩 / 에러 UX

### 첫 방문 Empty State

Nexi가 아주 짧게 "뭐부터 도와줄 수 있는지" 설명

### 분석 중 로딩

> "지금 공장/물류/관세 데이터들을 조합해서 '할 만한 게임인지' 보고 있어요…"

### 실패/불안정 케이스

**예**: 데이터 품질이 너무 낮거나, 카테고리 파악이 안 될 때

**메시지 예시**:

> "지금 정보로는 안정적인 숫자를 못 뽑겠어요. 아래 중 하나만 더 알려주실 수 있을까요?"

**옵션**:
- [타겟 판매 가격]
- [대략적인 예상 수량]
- [타겟 마진 %]

## 8. 리포트 / AI 분석 연동 방식

### 트리거

"1차 분석 시작하기" 버튼 → 백엔드 API 호출 (기존 app.nexsupply.net 분석 로직)

**API 엔드포인트**: `POST /api/analyze-product`

### 결과 표시

1. **채팅 버블로 요약** (핵심 3~5줄)
2. **"자세히 보기" 버튼** → 기존 리포트 화면 또는 모달

### 연속 사용

유저가 새 제품/새 옵션을 입력하면
같은 채팅 세션 안에서 2번째, 3번째 분석도 가능하게 설계

---

## 9. 결제 / 프로젝트 전환 플로우

### Deposit 설명 포인트

리포트가 마음에 들면, Nexi가 제안

> "이 프로젝트를 진짜 실행 모드로 바꾸고 싶으면, $10 Deposit만 걸어주세요. 상담 + 공장 매칭 + 포장/물류 설계까지 이어집니다."

### 결제 이후 상태 변화

- **유저 상태**: Lead → Project
- **채팅 상단에** "이 프로젝트는 이제 '진행 중 프로젝트'입니다" 같은 배지 표시
- **추후 사람(실제 팀) 개입 포인트 표시**

---

## 10. 데이터 / 상태 관리 메모 (개발 참고용)

### 최소 필요 상태

- `user_id`
- `project_id`
- 채팅 로그
- 수집된 구조화 필드 (채널, 국가, 카테고리, 가격 등)
- 분석 결과 snapshot

### 상태 저장 타이밍

- 특정 질문들에 답변이 들어올 때마다 `project_context` 업데이트
- 분석 실행 시점마다 버전 기록

---

## 11. 오픈 이슈 / 결정 필요 사항

1. **채팅 시작 시 로그인 필수 여부**
2. **다국어 지원** (영어/한국어 전환 UX)
3. **우측 패널을 MVP에서 포함할지, 나중에 추가할지**
4. **CRM(예: Notion, HubSpot 등)과의 리드 싱크 방식**
5. **Deposit 결제 플로우를 채팅 안에서 얼마나 깊게 통합할지**
